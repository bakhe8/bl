# تعليمات حرجة (أولوية قصوى)
- صفحة واحدة فقط، وكل التفاعلات داخل نافذة واحدة (modal): تحميل، تحذيرات، حل الغموض، معاينة/طباعة الخطاب.
- لا Frameworks (React/Vue/Angular/Bootstrap) ولا تقسيم لشاشات أو تبويبات؛ جافاسكريبت Vanilla فقط.
- واجهة المستخدم البسيطة: زر تحميل Excel → عرض التنبيهات/الأخطاء → جدول الصفوف الغامضة → زر إنشاء/معاينة خطاب.
- كل قرار ≤ 3 خطوات؛ أي خطوة إضافية تُرفض.
- لا ميزات غير موجودة في الوثائق/التعليمات؛ لا ابتكار خارج المتطلبات.
- الرسائل والتنبيهات والقرارات والخطاب كلّها في نفس المودال بأسلوب واحد.
- يعمل محلياً (بدون سيرفر) قدر الإمكان؛ إذا لزم، استخدم خادم محلي بسيط فقط لتجاوز قيود المتصفح.

# مرجع واجهة بسيط (من النموذج القديم لكن دون التزام بالتصميم/React)
- لوحة تنبيهات صغيرة تعرض الأخطاء/الصفوف الغامضة (عدد المختصر).
- جدول الصفوف التي تحتاج قرار (بنك/مورد خام، حالة يحتاج تأكيد).
- تفاصيل السجل المحدد (قراءة فقط).
- معاينة خطاب جاهز للطباعة لنفس السجل.
# تعليمات الربط
- هذه التعليمات لها الأولوية على أي شيء آخر في المشروع.
- يجب أن تُدمج مع فلسفة البساطة وقواعد الصفحة/المودال الواحدة.
