# Documentation

هذا الدليل هو المرجع الأساسي لمستندات المشروع ويجب تحديثه بشكل مستمر مع أي تغيير وظيفي أو معماري.

## فهرس الوثائق
- **نظرة عامة**: الرؤية، الأهداف، أصحاب المصلحة، ونطاق المشروع.
- **المتطلبات**: المتطلبات الوظيفية وغير الوظيفية، القيود، والافتراضات.
- **المعمارية**: مخطط عالي المستوى، المكونات الرئيسية، تدفق البيانات، والتبعيات الخارجية.
- **الإعداد والتشغيل**: خطوات تثبيت البيئة، الأوامر للتشغيل محلياً، وأي متطلبات مسبقة.
- **الاختبارات والجودة**: كيفية تشغيل الاختبارات، تغطية الكود، والسياسات المتبعة للجودة.
- **عمليات التطوير**: إرشادات الفروع، أسلوب الكود، مراجعات الكود، وإصدار النسخ.
- **إدارة المعرفة**: الأسئلة الشائعة، المشكلات المعروفة، وخطط التحسين.

## إرشادات التحديث
- كل ميزة جديدة أو تغيير معماري يجب أن ينعكس هنا فوراً.
- أضف روابط إلى الملفات التفصيلية داخل `docs/` كلما اتسعت المحتويات (مثلاً `docs/architecture.md`, `docs/setup.md`).
- حافظ على تواريخ وآخر تحديث لكل قسم عند الحاجة لضمان تتبع سليم.
