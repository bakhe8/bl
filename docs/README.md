# Documentation

هذا المجلد هو المرجع الرئيسي للمستندات. يتم تحديثه تلقائياً بناءً على تعليمات `docs/draft.md`. يرجى إبقاء المسودة محدثة بكل التغييرات.

## الفهرس
- **نظرة عامة**: `docs/overview.md` – التعريف، المشكلة، الحل، الرؤية، والأهداف.
- **المتطلبات**: `docs/requirements.md` – المتطلبات الوظيفية وغير الوظيفية، القيود، ونطاق العمل.
- **المعمارية**: `docs/architecture.md` – المكونات، تدفق البيانات، التخزين المحلي، والتقنيات المستخدمة.
- **الإعداد والتشغيل**: `docs/setup.md` – البيئة المقترحة، طريقة التشغيل، الاعتمادات، والنسخ الاحتياطي.
- **التشغيل اليومي**: `docs/operations.md` – سير العمل اليومي، إدارة القواميس، والمعالجة المتدرجة للحالات الغامضة.

## إرشادات التحديث
- أي تغيير في المتطلبات أو المعمارية أو التدفق التشغيلي يجب أن ينعكس في الملفات أعلاه.
- أضف ملفات تفصيلية جديدة داخل `docs/` عند الحاجة (مثل اختبارات، أمن، إدارة بيانات) مع تحديث الفهرس وروابطه.
- احتفظ بـ `docs/draft.md` كمكان مركزي لتجميع الملاحظات والقرارات قبل دمجها في الوثائق الرسمية.
