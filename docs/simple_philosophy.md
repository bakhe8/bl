# فلسفة البساطة – Simple First Development

1) الصفحة واحدة فقط داخل `index.html`؛ لا صفحات أو تبويبات أو تنقل أو sub-screens.
2) نافذة واحدة (modal) لكل شيء: قرار، خطأ، خطاب، طباعة؛ نفس الأسلوب ونفس الإطار.
3) ممنوع إضافة ميزات غير مذكورة في الوثائق/التعليمات؛ لا “إبداع” أو خيارات مستقبلية.
4) كل قرار ≤ 3 خطوات؛ أي ما يتجاوز ذلك يُعاد تصميمه.
5) لا إعدادات، لا قوائم كبيرة، لا شاشات جانبية؛ واجهة المستخدم = زر تحميل Excel + جدول الغموض + زر إنشاء خطاب.
6) تعقيد الداخل مسموح، تعقيد الخارج ممنوع: المنطق يمكن أن يكون معقداً، العرض يبقى بسيطاً جداً.
7) جاڤاسكريبت Vanilla فقط؛ لا Frameworks ولا مكتبات UI.
8) سرعة وبساطة الاستخدام أهم من “الكمال” التقني؛ أي محاولة تحسين زائد تُرفض.
