# وثيقة القواميس

القواميس هي قلب النظام لضمان تطابق أسماء البنوك والموردين بدقة ثابتة.

## أنواع القواميس
- **القواميس الرسمية**: `banks_official`, `suppliers_official` بصيغ عربية قياسية.
- **القواميس التعلمية (variants)**: تحتفظ بالأسماء البديلة المكتشفة وتحيلها إلى الصيغ الرسمية.

## التخزين
- LocalStorage هو المصدر التشغيلي.
- JSON للتصدير/الاستيراد (نسخة احتياطية أو مشاركة بين أجهزة).

## شكل بيانات مقترح (مثال)
```json
{
  "official": {
    "banks": ["بنك الرياض", "مصرف الراجحي"],
    "suppliers": ["شركة التوريد المتحدة"]
  },
  "variants": [
    { "type": "bank", "variant": "rajhi bank", "official": "مصرف الراجحي", "language": "en" },
    { "type": "supplier", "variant": "شركة توريد متحده", "official": "شركة التوريد المتحدة", "language": "ar" }
  ]
}
```

## إضافة/تحديث بنك أو مورد
- أضف الاسم الرسمي إلى القائمة الرسمية.
- اربط الـ variants المعروفة بالاسم الرسمي.
- عند اختيار المستخدم صيغة جديدة، تُحفظ كـ variant وتُستخدم تلقائياً لاحقاً.

## ترقية variant إلى رسمي
- إذا تكرر استخدام variant وتم اعتماده، يُنقل إلى القائمة الرسمية ويُحدّث كل الروابط.

## مبادئ
- لا تُحذف الـ variants السابقة إلا عند التأكد من إنها خاطئة.
- احتفظ بلغة/جهة الإدخال (عربي/إنجليزي) لتسهيل التطابق والبحث.
