# وثيقة خوارزميات التطابق

## طبقات التطابق
- تطابق مباشر مع القواميس الرسمية (عربي).
- تطابق باستخدام النسخ الإنجليزية أو الاختصارات.
- تطابق عبر الـ variants المتعلمة سابقاً.
- Fuzzy بسيط (اختياري) لتقليل الغموض عند تباين محدود.

## إدارة الغموض
- عرض الخيارات القريبة للمستخدم في شاشة القرارات.
- تأكيد المستخدم يُسجَّل كـ variant جديد لتطبيقه تلقائياً مستقبلاً.
- تجنب التغيير التلقائي إذا كانت الثقة منخفضة؛ أظهر الصف للمراجعة.

## حفظ القرار
- كل قرار (variant → official) يُحفظ في LocalStorage ويُصدَّر/يُستورد عبر JSON.
- احتفظ باللغات/الحالات لتجنب تصادم الأسماء المتشابهة.

## جودة التطابق
- تأكد من وجود حد أدنى للتشابه قبل اعتماد Fuzzy.
- اجمع السياق (مبلغ/تاريخ/رقم ضمان) للمساعدة في القرار اليدوي.
